I spent this week working on the terrain and perfecting the minutiae. Week 4: March 4, 2018

With impending midterms, this week was largely unproductive. I searched for more specific tutorials for Unity 3D to implement the rail track and cart, but I couldn’t find quality tutorials. Most of the videos I encountered were advertisements for expensive asset packages, or product display videos that were mislabeled as tutorials. Thankfully, Kyle was working on the same problem and was able to find an asset package that was not only free but he believes will work for our purposes.

The main part of the rail track and cart mechanism is making the cart follow a specific path (visually represented by the rail track) and having the camera follow the cart to give the first-person point of view. From my research, there are two general methods of approaching this type of problem. The first method involves using splines to create the curves in the track and attaching a cart to follow the spline. There were several videos I watched about this and it seemed fairly easy to implement, but the asset package used in those videos is outside the budget of this project (a.k.a more than $0). I then turned to the other method: using waypoints. While I haven’t dug into Kyle’s asset package yet, it appears waypoints are the method it uses.

During the team meeting this week, we discussed the Design page of the website. Since we use Google Drive to store our meeting notes and other group documents, we were all working on the same document to create the language for the Design page. I revised the Overview section, while the guys worked on the other two sections (Environment and Module Descriptions). After they were done, I copied all the text and images and imported it to the webpage. Clayton made the Module Description diagram, but after some suggestions from the group about color choices, he decided to remake it to improve clarity. I told him that since I already had the html document set up, he would only have to replace the image and make sure the images were named the same. There were some hiccups, but we got them resolved and the new diagram looks professionally done.

During the next two weeks, I will start taking Kelvin’s object models (tomb stones, rails, buildings, etc.) and start creating the terrain. I will start with a basic low-poly forested mountain range and create a cemetery on one side, then we can build from there. I’m looking forward to working on the terrain that will most likely appear in our final game release.

Week 3: February 25, 2018

This week I followed a tutorial (Camera on Rail) that ended up not being very helpful. I did learn some important lessons about how to make game objects and attach C# scripts to them. I thought it was going to be written in the code where to attach the C# script to the game object, but apparently, it’s just a drag-and-drop motion in the Inspector section of the menu. The reason the tutorial wasn’t helpful to me is because I couldn’t replicate the type of motion in the video in my own project. I’m not sure if it is because I didn’t place the reference sphere’s correctly, I messed up part of the C# script, or if it is a compatibility issue with the Unity version in the video and the Unity version I’m using (2017). I’m going to look up more tutorials about how to make a first-person game and how to automate the movement of a cart on a rail type system next week. Considering these are the main automated movements of our game, it’s important to find quality resources to learn from so that we can replicate the movement and be able to modify it to our purpose.

I started creating the designs for each level of the game. We decided to start with one level and, time pending, make more levels of increasing difficulty that the player can gain ‘points’ for defeating the monsters. One such level design included shooting ghosts. The player would shoot ghosts from a cart, accumulate points, and advance through different scenery along the way until the end of the level. For instance, the level would start in a funeral home, proceed through a cemetery, then move on to other places that would stereotypically inhabit ghosts (decrepit homes, abandoned mine tunnels, etc.). At the end of each level, the player swaps out their weapon to fight the next type of monster. In order to fight ghosts, the player needs ectoplasm guns, a werewolf is defeated with a wooden stake gun, etc. Each different type of monster will be a different level and each level will contain terrain that is typical of the monster being fought. These designs and mechanics will be further refined at a later date.
