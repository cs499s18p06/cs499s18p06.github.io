Week 3: February 25, 2018

This week I followed a tutorial (Camera on Rail) that ended up not being very helpful. I did learn some important lessons about how to make game objects and attach C# scripts to them. I thought it was going to be written in the code where to attach the C# script to the game object, but apparently, it’s just a drag-and-drop motion in the Inspector section of the menu. The reason the tutorial wasn’t helpful to me is because I couldn’t replicate the type of motion in the video in my own project. I’m not sure if it is because I didn’t place the reference sphere’s correctly, I messed up part of the C# script, or if it is a compatibility issue with the Unity version in the video and the Unity version I’m using (2017). I’m going to look up more tutorials about how to make a first-person game and how to automate the movement of a cart on a rail type system next week. Considering these are the main automated movements of our game, it’s important to find quality resources to learn from so that we can replicate the movement and be able to modify it to our purpose.

I started creating the designs for each level of the game. We decided to start with one level and, time pending, make more levels of increasing difficulty that the player can gain ‘points’ for defeating the monsters. One such level design included shooting ghosts. The player would shoot ghosts from a cart, accumulate points, and advance through different scenery along the way until the end of the level. For instance, the level would start in a funeral home, proceed through a cemetery, then move on to other places that would stereotypically inhabit ghosts (decrepit homes, abandoned mine tunnels, etc.). At the end of each level, the player swaps out their weapon to fight the next type of monster. In order to fight ghosts, the player needs ectoplasm guns, a werewolf is defeated with a wooden stake gun, etc. Each different type of monster will be a different level and each level will contain terrain that is typical of the monster being fought. These designs and mechanics will be further refined at a later date.

Week 2: February 18, 2018

This week came with a lot of studying, but not much work on the actual project. I created my ‘Hello World’ program in Unity3D, which allowed me to make sure my Unity installation was installed correctly and gave me a brief tutorial into C# programming. After completing my ‘Hello World’ program, I tinkered with the terrains and developed a rocky looking mountain. I learned how to use the six tools for manipulating the terrain. The first three tools were fairly intuitive to learn, but the last three required some research since I didn’t have any experience with image editing tools. I followed several tutorials to learn how to use the ‘Paint Texture’ tool, which taught me that you can scale a texture to blend it better across the entire terrain. I had fun painting on several layers of texture and blending them in order to make a rocky mountain range, but then I decided I needed trees and grass to decorate my mountains.

Learning how to add trees was interesting. I watched a tutorial that was difficult to follow because it was done on an older version of Unity3D and some of the features looked different or were in different places, but I was able to get the gist of what to do. I later found a more recent tutorial that explained things better and discovered many little intricacies to making terrains.

During the group’s weekly Friday meeting, we discussed the components of the website that had pending due dates over the weekend. We came up with a timeline of project goals that I added to our calendar.
