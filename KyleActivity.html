Week 4: March 4, 2018

As stated in the previous week's update stated, I spent the week working on the camera movement of the player as they move through the game environment. At first I tried using built in animations and other methods of movement within Unity 3D itself. With those methods not working, I discovered a wonderful asset called Camera Path Creator.

This plugin for Unity allows me to set the exact course for the camera to follow, using waypoints set where there camera should go. With these waypoints, it's easy for me to change how the camera moves, should another team member change the layout of the level. And with timing controls as well, this assures us that the camera the user operates will be exactly where we want it to be.

Week 3: Feburary 25, 2018

With the various parts of the application working on mobile device, I set to work on the actual mechanics of the game, rather than just getting a program up and running. At the moment, I am working on getting the camera to move with with the "cart" along the "track" for the experience. This is somewhat tougher than I originally thought, as the Cardboard plug-in uses a different camera than normal Unity, to allow the stereo 3D effect for virtual reality.

Additionally, I've been working on getting the shooting mechanic working for the game. As this is an "On Rails Shooter", it is a core component of the gameplay. One important thing for me in developing this is that it can be set up to shoot with the "clapping" coming from the microphone.
