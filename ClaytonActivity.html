Week 2: February 18, 2018

During this week I helped implement a few more designs and ideas for the team. For requirement section 2.3, I implemented the calendar and added a calendar section to the website. In order to complete this, I used our group’s Gmail account and embedded the calendar section into our website. We found this to be the most efficient use of a calendar because anybody on the team can update the calendar on any device that can access Gmail. Gmail also has a great built in UI experience so adding information to the calendar is easy and efficient.

For other parts of the website, I’ve included more information for the requirements page and the design consideration page.

For the project itself, I started getting a basic user interface being developed. I created a basic start menu that had a play, options, and quit button. When the cursor hovers over a button, the button becomes highlighted. With this, the user can go back and forth from the options menu to the main menu. The user can quit or the user can access the game.

I also developed a pause menu, same functionality as the start menu, but when the user selects the ESC key, the game is paused and the user is given access to the pause menu and the same types of functionality as the start menu.

However, implementing the two scenes together have been giving me some trouble. I believe this is an issue through Unity3D’s hierarchy system, and that this issue can be easily resolved with a little more research and troubleshooting. Once I can figure out how to go from screen to screen, I will start implementing VR user experience.

Lastly, I helped with the team decision making ideas on what game to create.
